# 报告查看变更功能说明

## 功能概述

在审核报告的每个文件详情中添加了"查看变更"按钮，点击后可以在模态框中查看该文件的完整变更内容（diff）?

## 实现方式

### 1. 数据结构修改 (internal/report/html.go)

?`FileReview` ?`FileReviewData` 结构体中添加了新字段?

```go

    FileName string
    Status   string
    Result   *ai.ReviewResult
    Error    error
    Revision int    // SVN版本号（在线模式?

```

### 2. 报告生成修改

#### 后端 (gui/server.go)

**本地模式?*
- 在审核循环中，获?diff 后保存到 `fileReview.Diff`
- 支持新增、修改、删除等不同状态的文件

#### 前端 (internal/report/html.go)

1. **HTML结构?*
   - 在文件头部添?查看变更"按钮

2. **样式?*
   - 绿色按钮样式 `.view-diff-btn`
   - 模态框样式 `.modal` ?`.modal-content`
   - diff 内容显示区域 `.diff-content`

3. **JavaScript功能?*
   - `viewDiff(index)` - 打开模态框显示指定文件?diff
   - `closeModal()` - 关闭模态框
   - 点击模态框背景也可关闭
   - 文件数据?JSON 格式嵌入?HTML ?

## 使用方式

### 在报告中查看变更

1. 打开审核报告（自动在浏览器中打开?
2. 找到要查看的文件
3. 点击文件头部?查看变更"按钮
4. 在弹出的模态框中查看完整的 diff 内容
5. 点击"关闭"按钮或模态框背景关闭

### 功能特点

- ?无需返回审核页面即可查看变更
- ?支持所有文件状态（新增、修改、删除）
- ?diff 内容以等宽字体显示，保持格式
- ?模态框支持滚动查看长内?
- ?点击背景或关闭按钮都可关闭模态框

## 与列表中"查看变更"的区?

### 列表中的"查看变更"
- 位置：在文件列表?
- 功能：实时从服务器获?diff
- 适用场景：审核前预览变更

### 报告中的"查看变更"
- 位置：在生成的报告中
- 功能：显示报告生成时保存?diff
- 适用场景：查看历史审核记?
- 优势：离线可用，无需服务?

## 注意事项

1. **报告大小?* diff 内容会增加报告文件大小，特别是对于大文件或多文件审核
2. **浏览器兼容性：** 使用了现代浏览器?CSS ?JavaScript 特?
3. **内容限制?* 非常大的 diff 可能影响报告加载速度
4. **编码问题?* 确保 diff 内容正确转义，避免破?JSON 格式
